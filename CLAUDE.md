# Pi Memorization Quiz — Project Guide

## What this is

A browser-based quiz that tests how many digits of π the user can recall from memory. The user types digits one at a time; the app shows a rolling tape of recent digits, tracks the count, and shows exactly where they went wrong.

## Who is working on this

Beginner developer. Keep explanations simple. Avoid complex abstractions or unnecessary dependencies.

## Tech stack

- Plain HTML, CSS, and JavaScript — no framework
- `pi_digits.js` — pre-generated array of 100,000 π digits (generated by `generate_pi.py`)
- Vite — dev server only (`npm run dev`), not used in the build output
- Deployed on **Netlify** (free tier)

## Deployment

The site is live on Netlify. The user has a **Netlify personal access token** for CLI/API access.

To deploy via Netlify CLI:
```bash
netlify deploy --prod
```

To log in using the personal access token:
```bash
netlify login
# or set env var: NETLIFY_AUTH_TOKEN=<token>
```

## Project structure

```
index.html        — entire app (HTML + CSS + JS in one file)
pi_digits.js      — const PI_DIGITS = [...] (100,000 digits)
generate_pi.py    — Python script to regenerate pi_digits.js
package.json      — only has vite as a dev dependency
```

## Running locally

```bash
npm install
npm run dev
# open http://localhost:5173
```

## Important notes

- `pi_digits.js` must be loaded before the main script. If missing, the app shows an error telling the user to run `generate_pi.py`.
- The app is entirely client-side — no backend, no database.
- `PI_DIGITS[0]` = '3', `PI_DIGITS[1]` = '1', etc. The UI pre-seeds '3' and the user starts typing from index 1.

## Feature ideas (everything possible)

### Persistence (localStorage — no backend needed)
- **Personal best** — save and display the user's highest score across sessions
- **Session history** — show a list of the last 10 scores
- **Total digits typed** — lifetime counter of all digits entered correctly
- **Daily streak** — track how many days in a row the user has practiced

### Gameplay modes
- **Speed mode** — measure how fast the user types correct digits (digits per minute)
- **Timed challenge** — how many digits can you get in 60 seconds?
- **Practice/peek mode** — show the next digit as a hint, but mark the attempt as practice
- **Hint button** — reveal the next digit on demand (with a penalty or separate score)

### Visual & UI improvements
- **Milestone celebrations** — confetti or flash animation at 10, 25, 50, 100, 500 digits
- **Light mode / color themes** — toggle between dark and light, or offer theme options
- **Digit grouping** — display the tape in groups of 5 (e.g. `14159 26535`) for readability
- **Digit position counter** — show "digit #47" alongside the count
- **Animated count** — smooth number roll-up animation when count increases
- **Progress bar** — visual bar showing progress toward next milestone

### Social / sharing
- **Share score button** — one click copies "I just got 47 digits of π!" or opens a tweet
- **Score card** — a styled image or text block the user can screenshot and share

### Gamification
- **Achievement badges** — unlock badges at milestones (e.g. "First 10", "Century Club")
- **XP and levels** — earn points per digit, level up over time
- **Combo multiplier** — score multiplies the longer your streak in a single session

### Educational
- **Pi facts** — show a random fun fact about π after each failed attempt
- **World record callout** — show how the user's score compares to famous memorizers
- **Digit breakdown** — after failing, show frequency of each digit in π (they're roughly equal)

### Accessibility & mobile
- **Larger tap targets** — bigger buttons for mobile users
- **Haptic feedback** — vibrate on correct/wrong (mobile browsers support this)
- **Font size toggle** — let users make the tape and counter larger

### Technical / PWA
- **Installable PWA** — add a `manifest.json` and service worker so users can install it on their phone like an app
- **Offline support** — service worker caches the app so it works without internet
- **Keyboard shortcut reference** — a small `?` button that shows all shortcuts

---

## What NOT to add (for now)

- A backend or database — not needed; localStorage covers all persistence
- A framework (React, Vue, etc.) — overkill for a single-page app this size
- User accounts / login — unnecessary complexity for a personal project
